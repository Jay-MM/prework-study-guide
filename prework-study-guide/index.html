HTML

  • HTML elements define the content and framework of a webpage
  • Usually have an opening tag and a closing tag represented with angle brackets that contain content and other nested elements.
  • Has both a head and body elements which contains information about the page which users can not see and the content of the page that users can see, respectively
  • The footer is always at the bottom of the page which usually contains the author, copyright, contact, sitemap, and navigation of the page.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • 3 ways to style a webpage using CSS are inline CSS, internal CSS style sheet, and external CSS style sheet.
  • To style an element directly in the HTML file, use inline CSS(inline CSS)
  • You can embed an entire stylesheet in the HTML file using a style element in the head element.(internal CSS style sheet)
  • using an external CSS style sheet is mostly preferred because it allows to keep all CSS rules in a separate file, which makes it easier to make changes to the design.
  • When using an external style sheet, link the file using a link element in the HTML's head

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git pull: fetches and merges any commits from the tracking remote branch
  • git push: Transmits local branch commits to the remote repo branch
  • git clone: retrieve an entire rep from a hosted location via URL
  • git add -A: commits ALL changes in the code
  • git commit -m "enter description here" : associates a message with commit, should be short and descriptive.

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • A group of related data is called an ARRAY.
  • the location of a data item in an array is a number called an index
  • You can comment out code by putting /* before and */ after a block of code
  • a FUNCTION is a set of instructions that tells the computer how to perform a certain task.
